## UCF Editor (Python): как быстро “чистить” UCF под конкретный проект

В шаблонном `Basys2_100_250General.ucf` много пинов (VGA, PS/2, PIO, EPP и т.д.).
Если проект использует только **SW/LED** (или ещё BTN/7seg), лишние ограничения в UCF часто вызывают предупреждения в ISE и мешают отладке.

Для этого в репозитории есть утилита **UCF Editor**, которая:

* открывает `.ucf`;
* показывает все `NET "..."` как **чекбоксы** (можно отключить ненужные связи);
* сохраняет новый `.ucf`, где отключённые строки **автоматически закомментированы**.

---

### Требования

* **Windows + Python 3.7+** (рекомендовано)
* Tkinter обычно идёт вместе с Python на Windows.
  Если при запуске ошибка про `tkinter`, переустановите Python с компонентом “tcl/tk”.

---

### Где лежит скрипт

Скрипт можно хранить прямо в папке проекта, например:

`C:\Xilinx\projects\ise-basys2-project-master\ucf_gui_py37.py`

---

### Запуск (Windows PowerShell)

Вариант 1 — перейти в папку и запустить:

```powershell
cd "C:\Xilinx\projects\ise-basys2-project-master"
py -3.7 .\ucf_gui_py37.py
```

Вариант 2 — запуск без перехода:

```powershell
py -3.7 "C:\Xilinx\projects\ise-basys2-project-master\ucf_gui_py37.py"
```

Если `py` отсутствует:

```powershell
python "C:\Xilinx\projects\ise-basys2-project-master\ucf_gui_py37.py"
```

---

### Как пользоваться

1. Нажмите **Open / Открыть .ucf** и выберите исходный UCF (например, `Basys2_100_250General.ucf`).
2. Используйте пресеты:

   * **Preset: только SW+LED** — для проектов типа P01/P02
   * **Preset: SW+LED+BTN** — если используются кнопки
3. При необходимости вручную включайте/выключайте отдельные `NET`.
4. Сохранение:

   * **Overwrite** — перезаписать текущий открытый `.ucf`
   * **Named** — создать новый файл `<ProjectName>.ucf` в папке проекта
   * **Save As…** — выбрать путь вручную

⚠️ Рекомендация: безопаснее сохранять как **новый файл** `<ProjectName>.ucf`, чтобы не “ломать” общий шаблон для других проектов.

---

### Подключение нового UCF в Xilinx ISE

Если вы сохранили **новый** файл UCF:

1. `Project → Add Source…`
2. выберите ваш `<ProjectName>.ucf`
3. удалите/отключите старый UCF из проекта, чтобы не было конфликтов по LOC.

Если вы выбрали **Overwrite** (перезаписали старый UCF) — ничего добавлять не нужно, просто пересоберите проект.

---

### Зачем это нужно (типовые эффекты)

* Убирает предупреждения из-за лишних `inout` линий (PIO/EppDB/PS2), если они не используются.
* Убирает конфликты по VGA (если в UCF есть PULLUP, а в VHDL вы держите VGA в 0).
* Делает проект “чистым”: в UCF остаются только реально используемые пины.

---

### Примечание про “Unused IOB Pins” (ISE)

Если вы выкинули 7-seg из UCF/портов и видите паразитное свечение индикаторов, проверьте:
`Generate Programming File → Process Properties → Unused IOB Pins`

* **PullDown** может включать активный-low 7-seg (паразитная подсветка)
* **Float** часто полностью гасит индикаторы (пины в Hi-Z)
